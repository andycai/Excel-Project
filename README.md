# Excel-Project
一个使用 Excel 来做项目管理和缺陷管理的小插件。

因为很多研发团队还是习惯使用 Excel 来写文案，写计划，写需求，反馈缺陷等，很难完全改变去使用想 Teambition、Worktile和石墨工具，或者开源的一些 Web 的项目管理工具（可以自己内网搭建），就是觉得 Excel 用起来特别的“爽”，既然不改变工具，那就让现有的工具用得更方便和更爽吧，于是就写了这个小插件。

原本的初衷是为了解决 Excel 文件版本管理合并困难的问题，使用 vba 写的一个小插件。使用这个小插件，每个人都只需要管理好自己 Excel 文件，不需要去修改其他人的文件，其他人文件有更新，只需要运行一个宏就可以拉取到最新的内容，最棒的是主管可以使用这个插件来做各种统计，例如现在还有哪些工作没有完成，现在还有哪些工作反馈是完成困难的，现在完成和未完成的工作数量和比例，每个部门的每个人的完成任务的情况等等，是不是很棒？


# 流程图

开始 ->
		提出需求 ->
		拉取需求任务 ->
		完成任务 ->
		拉取完成后任务 ->
		验收任务 ->
			if 验收完成 -> Goto 结束
			else 反馈任务 -> Goto 开始
结束

# 示例

这里有一个完整的示例，包括技术、策划和美术三个目录，还有两个文件，修正任务和模板文件。

我们尝试运行一下示例，首先，进入“技术”目录，使用 Excel 打开里面的“白居易.xlsm ”文件，从“开发工具”菜单中点击“宏”，弹出的菜单中选中“拉取数据”，你会看到已经从策划的目录中拉取了有关于“白居易”的数据，同样，你可以进入“美术”目录，进行同样的操作，也能够拉取到属于“李清照”的数据。然后在“白居易.xlsm”文件中，在拉过来的数据中，对一些数据的完成情况进行备注，例如把1,3条数据标注为已完成，保存后，进入策划的目录，同样打开“杜甫.xlsm”文件，同样的操作拉取数据，你会看到刚才标注的完成的任务的状态都同步过来了。

这个示例说明了，这个Excel-Project的基本用法，本质就是各部门的同事直接任务的管理，流程大概如下：

策划提交了任务需求（或者是bug），技术和美术会拉取新的任务过来，完成后标注完成情况，然后策划把完成的任务拉取回来进行验收，验收完成后就标注完成，然后归档。其实流程是比较简单，要想有更多的需求，可以看看每个excel文档中的“说明”sheet，里面有详细说明参数配置，可以尝试着修改里面的参数，然后拉取数据看看结果。

# 配置

第一个 Sheet “说明”为配置，A2 里面的内容就是你要选择的配置方案，如果填 B，就是选择B列作为你当前的配置，配置列可以任意增加，只要你填上需要选择的列就可以。

# 策划目录

需求内容的生产者，次策划目录中文件提交产品需求，bug反馈等内容，后需要等带技术和美术来完成需求，然后把完成的需求更新回来，验收完成后，流程就结束了。

# 技术目录

需求的处理者，技术和美术目录的文件是处理策划提交的需求，第一步是需要把策划的需求同步过来，然后处理完成后，标注完成，然后策划能够同步完成的内容后，验收。

# 美术目录

同技术目录。

# 模板文件

模板文件.xlsm，任何人都可以从模板文件拷贝一份出来，修改一些配置，然后作为自己的管理文件。

# 修正任务

有些任务指定的负责人如果有误，或者是临时需要调整负责人时，如果不想去修改其他人的文件的话或者跑去告诉其他人需要修改一下，这里有一个方法可以做到，就是修正文件.xlsx这个文件，把需要修正负责人的任务添加进去，那么任务就会悄悄地修正为这里的负责人。
